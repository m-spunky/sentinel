## PS1

"Develop an AI-powered system capable of detecting phishing attempts in real time from email content or URLs."
Detailed Context
Phishing remains one of the most common cyber threats. Modern attacks use AI-generated content and URL obfuscation to trick users. Traditional systems rely on static blacklists which fail to detect zero‑day phishing campaigns. The goal is to design an ML-based system that analyzes email text/URLs and provides a confidence score.

Prototype Scope
•Accepts email text or URLs as input
•Uses ML/NLP to classify phishing vs legitimate content
•Generates a phishing probability score
•Displays results through a simple dashboard

Evaluation Metrics
- Detection Accuracy
- False Positive Rate
- Detection Speed
- Explainability
- User Experience


## PS2
"Design an AI-powered system that detects automated attacks such as credential stuffing and bot traffic in FinTech applications."

Detailed Context
FinTech platforms handle sensitive financial data. Attackers use bots to mimic human behavior for brute-force logins and API abuse. The objective is to build a system that analyzes login activity and application traffic logs to detect suspicious automated behavior.

Prototype Scope
•Analyzes login attempt logs and user activity
•Detects abnormal patterns from specific IPs
•Identifies bot-like behavior using anomaly detection
•Displays suspicious attack patterns on a dashboard

Evaluation Metrics
- Attack Detection Accuracy
- Real-Time Detection
- False Positive Rate
- Automation Capability
- Scalability

## PS3
"Develop an AI-powered system that detects fraudulent financial transactions using machine learning and anomaly detection techniques."

Detailed Context
Digital payment platforms face threats from account takeovers. Traditional systems rely on static rules which struggle with evolving fraud. This challenge focuses on building a system that identifies patterns and assigns risk scores before transactions are fully processed.

Prototype Scope
•Analyzes transaction datasets in real-time
•Detects unusual transaction patterns
•Assigns a fraud risk score to each transaction
•Displays suspicious transactions in a dashboard

Evaluation Metrics
- Fraud Detection Accuracy
- False Positive Rate
- Detection Latency
- Adaptability
- Explainability

## PS4
"Create an AI-powered chatbot that helps security teams analyze security logs and investigate potential threats using natural language queries."

Detailed Context
Security analysts monitor massive volumes of data. An intelligent chatbot interface can help analysts query security logs (failed logins, abnormal IP behavior) quickly to obtain insights without manual log parsing.

Prototype Scope
•Uses security log datasets (Network/Login activity)
•Allows users to query logs using natural language
•Returns summarized security insights or alerts
•Highlights suspicious activity like repeated failures

Evaluation Metrics
- Query Understanding
- Response Accuracy
- Threat Insight Generation
- User Experience
- Performance

## PS5
"Design an AI-powered system that monitors dark web sources and identifies potential cybersecurity threats."

Detailed Context
The dark web hosts communities where criminals trade stolen credentials and malware. Many organizations fail to detect these hidden threats until it is too late. This challenge seeks an NLP-based system to analyze forum text and identify mentions of leaked data.

Prototype Scope
•Analyzes text from forums or leaked datasets
•Identifies mentions of organizations/credentials
•Detects suspicious patterns using NLP models
•Generates alerts and threat insights via dashboard

Evaluation Metrics
- Threat Detection Accuracy
- Threat Insight Quality
- NLP Effectiveness
- Scalability
- Usability
